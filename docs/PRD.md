# 积分与酒小程序 PRD

## 产品目标

小程序包含两条主线：

- 积分申请、喝酒申请与审批闭环
- 酒款内容浏览与互动

目标是把积分、审批、消息提醒、酒类内容都放到同一个微信小程序里完成。

## 角色

- `USER`：提交加分申请、喝酒申请，查看积分、申请记录、酒百科内容
- `APPROVER`：审批与自己绑定用户的申请，查看待审批和审批历史
- `ADMIN`：维护用户角色，巡检审批关系
- `SOMMELIER`：维护酒款内容

## 核心业务规则

### 积分

- 积分变动只通过审批通过后写入积分流水实现
- 加分申请审批通过后增加积分
- 喝酒申请审批通过后扣减积分
- 审批人可通过首页快捷入口直接对当前绑定的被审批对象执行加分或减分
- 审批拒绝或申请撤回都不会扣减积分
- 直接调分写入积分流水，来源标记为 `manual_adjust`
- 直接调分允许被审批对象余额为 0 或负数，并允许调分后余额继续为负数

### 喝酒申请

- 喝酒申请由申请人自行填写本次消耗积分 `cost_points`
- 提交时会先校验当前积分余额是否足够
- 同一用户同一时间仅允许 1 条待审批喝酒申请
- 扣分时以申请单中保存的 `cost_points` 快照为准

### 审批关系

- 普通用户与审批人是一对一关系
- 用户可邀请另一位用户成为自己的审批人
- 审批人接受后建立关系
- 只有当前绑定的审批人可以审批该用户后续提交的申请
- 若当前关系下仍存在待审批申请，双方都不能直接解绑

### 审计与通知

- 申请创建、审批、撤回、解绑等关键动作写入 `operation_log`
- 审批通过/拒绝、邀请、解绑等事件写入 `notification`
- 审批人直接调分也会写入 `operation_log` 与 `notification`

## 首页规则

- "当前积分"卡片点击后进入积分流水
- "待我审批"卡片点击后进入待审批列表
- "未读消息"点击后进入消息中心
- 审批人若存在被审批对象，快捷入口展示"直接调分"
- 首页不再展示"当前积分进度"
- 首页不再展示"参数配置"入口

## 酒款模块

- 酒款内容由 `wine_topic` 集合维护
- 用户可评分、留言（可独立提交，互不强制）
- 同一用户对同一酒款仅保留 1 条评价记录，再次提交时更新原记录
- 点击星级可独立更新评分，不要求先提交留言
- 品酒师可维护酒款名称、类别、风味、酒精度、口感等级、介绍、封面
- 口感等级：酸、甜、苦、辣各5级，具体描述如下：

| 口感 | 1级 | 2级 | 3级 | 4级 | 5级 |
|------|-----|-----|-----|-----|-----|
| 酸 | 几乎无酸 | 微酸清新 | 酸度平衡 | 酸感明显 | 酸爽突出 |
| 甜 | 几乎不甜 | 微甜柔和 | 甜度平衡 | 甜感明显 | 香甜浓郁 |
| 苦 | 几乎不苦 | 微苦回甘 | 苦度平衡 | 苦感明显 | 苦韵突出 |
| 辣 | 几乎不辣 | 微辣顺口 | 辛感适中 | 辛辣明显 | 烈感强劲 |

## 页面结构

### 底部导航

- 首页
- 酒
- 记录
- 我的

### 主要页面

- 首页：个人信息、积分、待审批、快捷入口
- 直接调分：审批人查看当前被审批对象并直接执行加分/减分
- 酒列表：酒款卡片（点击图片进入详情）
- 酒详情：酒款信息、评分、留言
- 记录：工作备忘录，按日期筛选待办
- 历史申请：加分申请和喝酒申请的历史记录
- 我的：资料维护、审批关系、邀请审批人
- 待审批列表：审批人查看自己负责的申请
- 审批历史：审批人查看历史审批记录
- 角色管理：管理员维护角色
- 酒款维护：品酒师维护酒款内容

## 2026-03-04 补充

### 待办工作业务规则

- 待办创建后状态为「待完成」，需手动点击标记完成
- 普通工作：点击完成后直接标记为「已完成」
- 加分工作：点击完成后提交审批，状态变为「待审批」，审批通过后入账
- 已完成的待办可重新打开或删除
- 待完成状态的待办可编辑内容
- 支持按日期筛选待办，默认显示今日

### 待办状态流转

| 状态 | 说明 |
|------|------|
| `todo` | 待完成 |
| `completed` | 已完成（普通工作） |
| `pending` | 待审批（加分工作） |
| `approved` | 审批通过 |
| `rejected` | 审批拒绝 |

### 记录页

- 以备忘录形式展示待办工作
- 默认显示全部待办，支持日期筛选
- 优先展示未完成的待办，其次按时间倒序
- 显示所有未完成待办数量
- 显示创建日期和时间、完成时间
- 所有状态均可点击查看详情
- 日期筛选使用中国时区（UTC+8）
