# 积分与酒小程序 PRD

## 产品目标

- 把积分、审批、消息提醒放到同一个小程序里完成
- 把酒款内容浏览、互动和个人记录整合到同一套体验里

## 角色

- `USER`：提交申请、查看积分、浏览酒款、维护个人记录
- `APPROVER`：审批绑定用户的申请与加分工作
- `ADMIN`：维护用户角色
- `SOMMELIER`：维护酒款内容

## 核心规则

### 积分

- 加分申请审批通过后增加积分
- 喝酒申请审批通过后扣减积分
- 审批人可通过首页快捷入口直接调分
- 直接调分写入 `points_ledger`，来源为 `manual_adjust`
- 直接调分允许当前余额和调整后余额为负数

### 待办工作

- 待办分为普通工作和加分工作
- 普通工作完成后直接标记为完成
- 加分工作完成后提交审批，审批通过后入账积分
- 未绑定审批人时，前端不允许开启加分工作
- 记录页列表只展示标题与标签，不展示状态文字与时间

### 审批关系

- 普通用户与审批人是一对一关系
- 用户可邀请另一位用户成为自己的审批人
- 若当前关系下仍有待审批申请，双方不能直接解绑

### 酒款模块

- 酒款由 `wine_topic` 集合维护
- 用户可收藏、评分、留言
- 同一用户对同一酒款仅保留 1 条评价记录
- 酒详情展示封面、名称、类别、度数、介绍、风味、口感简图、基酒与原料、口感解读、背景故事、相似推荐
- 相似推荐从现有酒款中选择，最多 3 个，并按风味相近程度排序
- 酒款维护当前字段包含：名称、类别、酒精度、风味、口感等级、基酒、原料、口感解读、背景故事、相似推荐、封面

## 首页规则

- “当前积分”点击后进入积分流水
- “待我审批”点击后进入待审批列表
- “未读消息”点击后进入消息中心
- “未读消息”展示在昵称同一行右侧
- “当前积分”“待我审批”采用“标题：数值”的紧凑形式
- 审批人存在被审批对象时，快捷入口显示“直接调分”
- 首页展示内嵌喝酒日历

## 页面结构

- 首页
- 酒
- 记录
- 我的

主要页面：

- 首页：个人信息、积分、待我审批、快捷入口、喝酒日历
- 直接调分：审批人查看当前被审批对象并直接调分
- 酒列表：点击图片进入详情
- 酒详情：收藏、评分、留言、相似推荐
- 记录：待办列表、日期筛选、详情
- 我的：修改资料、审批关系、邀请审批人、我的收藏
- 酒款维护：品酒师后台
